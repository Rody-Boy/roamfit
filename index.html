RoamFit is a credit-based fitness marketplace connecting members with independent gyms and fitness facilities. This GitHub Pages site is the public landing page while the Laravel MVP application is prepared for PHP hosting.

Laravel MVP build path

The production app is now a single Laravel MVP instead of a complex multi-app scaffold. The first release focuses on the core marketplace loop.

Home

Laravel route: /

  • Shows the RoamFit tagline.
  • Highlights featured active facilities.
  • Has CTAs to browse facilities or create an account.

Facilities

Laravel route: /facilities

  • Lists only facilities with active status.
  • Search supports keyword, city, and province.
  • Each card shows location, categories, and credit cost.

Facility Detail

Example Laravel route: /facilities/demo-fitness-manila

  • Shows description, city/province, credit cost, capacity, and status.
  • Logged-out users see a login-to-check-in CTA.
  • Check-in workflow starts in Phase 3.

Register

Laravel route: /register

  • Creates a member account.
  • New users default to role member and status active.

Login

Laravel route: /login

  • Authenticates existing users.
  • Seeded admin is admin@roamfit.test / password after local seeding.

Dashboard

Laravel route: /dashboard

  • Requires login.
  • Shows member credit balance.
  • Shows recent credit ledger entries.

Admin

Laravel route: /admin

  • Requires an authenticated admin user.
  • Links to businesses, facilities, and categories.

Admin Businesses

Laravel route: /admin/businesses

  • Create and edit partner businesses.
  • Assign owner users.
  • Set status: pending, approved, rejected, or suspended.

Admin Facilities

Laravel route: /admin/facilities

  • Create and edit facility listings.
  • Assign business, categories, city/province, status, capacity, and credit cost.
  • Only active facilities appear publicly.

Admin Categories

Laravel route: /admin/facility-categories

  • Create configurable facility categories.
  • Enable/disable categories.
  • Sort categories for admin/user display.

Credits

Current code: migrations/models for credit_packages, purchases, and credit_ledger_entries.

  • Admin credit package screens are planned for Phase 2.
  • Manual purchase confirmation comes before payment gateway integration.

Check-ins

Current code: migrations/models for check_in_tokens and check_ins.

  • Member token generation and staff redemption are planned for Phase 3.
  • The database primitives are already present for the MVP loop.

GitHub Pages can only serve static files. The Laravel app in this repository requires a PHP host such as Laravel Forge, Ploi, Render, Railway, DigitalOcean, or another server that can run PHP and a database.
